Ghi chú:
- trong cặp dấu <[]> là câu lệnh
- trong cặp dấu <{}> là thay đoạn trong đó bằng đoạn khác tương ứng

**** TẢI 1 REPOSITORY VỀ MÁY
Bước 1: tạo 1 repository trên github (có file readme)
Bước 2: tạo một folder trên desktop, chuột phải, chọn <[git bash here]>
Bước 3: gõ lệnh <[git clone <{sao chép đường dẫn http của repository(chỗ clone 
or download trên github}>  ]> để tải về repository đó trên folder vừa tạo 
trên desktop.
**** ĐƯA CÁC FILE LÊN GIT
Bước 1: chuột phải trong thư mục chứa file cần đưa lên github, chọn git bash here
Bước 2: gõ lệnh <[git status]> để xem có file nào chưa được git và theo dõi không
(untracked files) (hay có file thay đổi)
Bước 3: nếu muốn theo dõi file đó gõ lệnh <[git add <{tên file hoặc để dấu * nếu
muốn lấy tất cả file}>]>
Bước 4: chạy lại bước 2, thay vì báo đỏ như bước 2, thì nó báo xanh (tức nó báo 
có 1 file mới). Tuy nhiên lúc này, nó chưa được lưu trữ vào kho chứa local của 
chúng ta. Để làm điều đó thực hiện bước 5.
Bước 5: gõ lệnh <[git commit -m"<{đưa vào dòng comment}>"]> tức là xác nhận, chạy
xong nó sẽ báo có một file đã thay đổi. Tuy nhiên lúc này nó mới chỉ nằm trong
kho chứa local của chúng ta. Muốn github được đồng bộ với local của chúng ta (hay
cập nhật lên github) thực hiện bước 6.
Bước 6: gõ lệnh <[git push]> (nếu lần đầu push lên, sẽ có yêu cầu email và mật
khẩu)
** có thể gộp lệnh add và commit thành lệnh:
	<[git commit -a -m"<{đưa vào dòng comment}>" ]>
--------------------------------------------------------------------------
**** BACK PROJECT TẠI MỘT THỜI ĐIỂM TRONG QUÁ KHỨ
Bước 1: gõ lệnh <[git log]> để lấy các id của những lần update, chuột phải
và copy id của lần cần quay lại.
Bước 2: gõ ctrl+c để dừng lệnh gốc lại và chuẩn bị cho những câu lệnh khác
Bước 3: gõ lệnh <[git revert <{id vừa copy}> ]> để quay trở lại thời điểm 
muốn backup.
Bước 4: enter, nhấn esc, nhấn shift, dấu hai chấm, gõ <[q]>, enter. Vậy là đã 
quay trở lại với màn hình trước.
Bước 5: gõ lệnh <[git status]> để xem có cần commit lại không, nếu có dòng
working tree clean là đã commit thành công, không cần phải commit lại. Vậy
là đã quay trở lại thời điểm ta cần.
Bước 6: gõ <[git push]> để đồng bộ hóa với git hub
**** TRƯỜNG HỢP ĐỒNG NGHIỆP THAY ĐỔI FILE CỦA CHÚNG TA NGAY TRÊN GITHUB
Để đồng bộ hóa những file của chúng ta với những file của đồng nghiệp:
Bước 1: gõ lệnh <[git pull]> để nó cập nhật local của chúng ta với trên github
